Flink ML : Beispiel NaiveBayes

  • Naive Bayes ist ein Beispiel für einen Klassifikationsalgorithmus

  • d.h. Ziel ist es, eine Funktion zu finden, die einem gegebenen Input eine Kategorie (Klasse) zuordnet

  • Beispiel für Klassifikationsaufgabe: Reparatur von Zügen

    • Input: Schadensbericht durch das Zugpersonal (beschreibt Symptome des Problems)

    • Output: (potenziell) beschädigtes Maschinenteil (beschreibt Ursprung des Problems)

    • → wenn dies durch einen Algorithmus mit einer hohen Zuverlässigkeit erledigt werden kann, spart es dem technischen Fachpersonal Zeit

  • Eine Schwierigkeit hierbei: Input sollte in einem einheitlichen Format vorliegen (Text ist ungeeignet) und möglichst nur für die Aufgabe relevante Information (Features) enthalten

    • → Feature Engineering

  • Naive Bayes ist eine einfache, ältere Methode, die auf Wahrscheinlichkeitsrechnung basiert, aber dennoch in manchen Anwendungen erstaunlich effektiv sein kann

Flink ML : Beispiel NaiveBayes (2)

  • Wir gehen davon aus, dass die Features eines Inputs durch einen Vektor einer fixierten Länge n gegeben werden:

    • (x_1, .. , x_n)

    • Hierbei kann jedes der n Features nur eine endliche Anzahl von Werten annehmen (z.B. eine ganze Zahl in einem begrenzten Bereich)

      • → ggf. sind die x_i bereits Kategorien für Features (z.B. Zahlbereiche)

  • Seien C_1, .., C_k die möglichen Klassen für die Klassifizierung

  • Bezeichne jeweils p(C_i | x_1,..,x_n) die Wahrscheinlichkeit, dass die Klasse C_i korrekt ist für den Inputvektor (x_1, .., x_n)

  • Dann findet ein Naive Bayes Klassifikator bei gegebenen Inputs x_1,..x_n denjenigen Index i, für den diese Wahrscheinlichkeit p(C_i | x_1,..,x_n) maximal wird

  • Damit dieses i praktisch effektiv ausgerechnet werden kann wird folgende ("naive") Annahme getroffen:

    • Die Werte der Features x_i und x_j in den Inputs für unterschiedliche i und j sind immer von einander statistisch unabhängig

Flink ML : Beispiel NaiveBayes (3)

  • Diese Annahme ist fast immer falsch, aber vereinfacht die Rechnung stark

    • Der Algorithmus funktioniert besser in Szenarien, in denen die tatsächliche Abhängigkeit der Features gering ist

  • Der Algorithmus benötigt in der Trainingsphase Beispiele für korrekte Klassifizierungen

    • → "supervised learning" (ML Paradigma)

    • Die vom Algorithmus angenommenen Wahrscheinlichkeiten basieren auf den Häufigkeiten der Kombinationen von Featurewerten und Klassen in den Trainigsdaten

    • Das Modell führt eine Art statistische Interpolation durch, um noch nicht gesehenen Featurekombinationen eine Klasse zuzuordnen

  • Der Algorithmus erhält seinen Namen aus dem Satz von Bayes über bedingte Wahrscheinlichkeiten:

    • p(C_i | x) * p(x) = p(x | C_i) * p(C_i)

      • hierbei ist x ein Featurevektor x = (x_1, .., x_n)

Flink ML : Beispiel NaiveBayes (4)

  • p(C_i | x) * p(x) = p(x | C_i) * p(C_i)

    • über die Trainigsdaten ist p(C_i) bekannt (relative Häufigkeit der Klasse C_i)

    • aus der Unabhängigkeit der Features ergibt sich:

      • p(x | C_i) = p(x_1 | C_i) * p(x_2 | C_i) * …​ * p(x_n | C_i)

    • die Werte p(x_j | C_i) sind wieder als relative Häufigkeiten den Trainingsdaten zu entnehmen

    • Der Wert von p(x) spielt für die Bestimmung des Maximalwertes von p(C_i | x) keine Rolle

  • → der maximierende Index i kann effektiv ermittelt werden, wenn der Algorithmus alle möglichen Werte von p(x_j | C_i) und p(C_i) tabelliert hat

  • Die Anzahl der initial zu berechnenden Basiswahrscheinlichkeiten ist in etwa gleich dem Produkt aus

    • (Anzahl der Features n) * (Anzahl der möglichen Werte eines Features) * (Anzahl der Klassen k)

  • Diese entsprechen der Model Data eines NaiveBayesModel in Flink

Flink ML : Beispiel NaiveBayes (5)

  • genauer werden die Basiswahrscheinlichkeiten logarithmisch transformiert

    • damit muss für die Berechnung des Maximums für jede Klasse i statt eines Produkts nur eine Summe gebildet werden (effizienter)

  • zusätzlich wird noch eine Technik namens Laplace Smoothing angewendet, um zu vermeiden, dass die Wahrscheinlichkeit p(x | C_i) gleich 0 wird, wenn einer der Faktoren

  • p(x_1 | C_i), ..,p(x_n | C_i) gleich 0 ist

    • Dies kann passieren, wenn ein möglicher Featurewert in den Trainingsdaten stark unterrepräsentiert ist

    • zudem ist der Logarithmus von 0 nicht definiert

  • Die Werte p(x_j | C_i) werden so transformiert, dass das Resultat nicht 0 werden kann (und dann logarithmisiert)

  • Der Algorithmus entspricht damit nicht mehr exakt Naive Bayes, ist aber in der Praxis robuster

  • Für eine Beispielanwendung von NaiveBayes in der API siehe eine der vorherigen Folien !

Flink ML : Beispiel KMeans

  • KMeans ist ein Beispiel für einen Clustering-Algorithmus

    • d.h. Ziel ist es, eine Menge von n Input-(Zahl)vektoren in k disjunkte Teilmengen (Cluster) zu partitionieren

    • hierbei sollten nah beieinander liegende Vektoren möglichst im gleichen Cluster gruppiert werden

  • Clustering ermöglicht eine Art Klassifizierung ohne die Notwendigkeit für zusätzliche Trainingsdaten

    • → "unsupervised learning" (ML Paradigma)

  • Anwendungen z.B. in Data Mining, Bildsegmentierung, Analyse von Benutzerverhalten, ..

Flink ML : Beispiel KMeans (2)

  • Für den KMeans Algorithmus wird eine positive ganze Zahl K als Parameter übergeben, dies ist die Anzahl der Cluster

  • Eine Optimallösung für KMeans findet K Clusterschwerpunkte (means) und ordnet jeden Inputvektor dem nächstgelegenden Schwerpunkt zu, sodass die Gesamtsumme der Abstände aller Inputvektoren zu ihren zugewiesenen Schwerpunkten minimal wird

    • hierbei wird der Abstand in der euklidischen 2-Norm gemessen, d.h. der Abstand der Vektoren ist die Summe der quadrierten Abstände der Vektorkomponenten

  • Eine Optimallösung zu finden, ist jedoch zu aufwändig (NP-schweres Problem), daher werden in der Praxis nur approximative Lösungen berechnet

Flink ML : Beispiel KMeans (3)

  • In Flink ML ist eine Version des iterativen Lloyd-Algorithmus implementiert (mit Hilfe der Iterations API) :

    • hierbei werden zunächst K zufällige initiale Clusterzentroide unter den Inputs gewählt gewählt

    • dann werden diese bei jeder Iteration verbessert:

      • jeder Inputvektor wird seinem nächstliegenden Zentroid zugeordnet

      • jeder Zentroid wird ersetzt durch den Schwerpunkt (Durchschnittsvektor) aller ihm zugeordneten Punkte

    • wiederhole die Iteration eine festgelegte Anzahl von Malen (konfigurierbar, default ist 20)

  • In einer KMeansModelData werden die berechneten Zentroide gespeichert

    • Anwendung des Models auf einen Vektor ordnet diesem den nächstgelegenden Zentroid zu

  • Es gibt auch noch eine Variante "OnlineKMeans", bei der mit jedem Zentroid Gewichte gespeichert werden, und es möglich ist, kontinuierlich neue Inputdaten hinzuzufügen, wodurch das Model kontinuierlich angepasst wird

    • → "online learning" (ML Paradigma)
